     sseettllcciidd -- set login context

SSYYNNOOPPSSIISS
     ##iinncclluuddee <<ssyyss//llccttxx..hh>>

     _i_n_t
     sseettllcciidd(_p_i_d___t _p_i_d, _p_i_d___t _l_c_i_d);

DDEESSCCRRIIPPTTIIOONN
     The sseettllcciidd() system call sets the login context of the specified process
     _p_i_d to the specified _l_c_i_d.  If the _p_i_d is zero, then the call applies to
     the current process.  If the _l_c_i_d is zero a new login context will be
     created.  If the _l_c_i_d is -1 the process will be removed from the login
     context it is currently a member of, if any.

     Creation of a new login context is only valid for the current process.  A
     process may not create a new login context if it is currently a member of
     one.

     Superuser privilege is required to add or remove a process from a login
     context.

RREETTUURRNN VVAALLUUEESS
     The sseettllcciidd() function returns the value 0 if successful; otherwise the
     value -1 is returned and the global variable _e_r_r_n_o is set to indicate the
     error.

EERRRROORRSS
     The sseettllcciidd() function will fail if:

     [EPERM]		Operation not permitted.

     [ESRCH]		No such process.

     [ENOMEM]		Cannot allocate memory.

     [ENOATTR]		Attribute not found.
